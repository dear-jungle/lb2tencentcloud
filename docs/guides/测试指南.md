## 测试类型

### 1. 单元测试 (Unit Tests)

测试单个模块或函数的功能，使用 Mock 隔离外部依赖。

**标记**：`@pytest.mark.unit`

```python
@pytest.mark.unit
@pytest.mark.aliyun
class TestAliyunService:
    def test_list_instances(self, mock_aliyun_client):
        """测试拉取阿里云 CLB 实例列表"""
        mock_aliyun_client.return_value = {
            'LoadBalancers': {'LoadBalancer': [mock_lb_data]},
            'TotalCount': 1
        }
        result = service.list_instances('cn-shanghai')
        assert len(result) == 1
```

**覆盖范围**：
- 阿里云服务：配置读取、数据转换
- 腾讯云服务：配置读取、写入操作
- 配置映射引擎：协议映射、调度算法映射、健康检查映射、不兼容检测
- 迁移引擎：状态机、断点续传、暂停/继续
- 报告生成：统计汇总、导出格式

### 2. 集成测试 (Integration Tests)

测试多个模块协同工作，验证端到端流程。

**标记**：`@pytest.mark.integration`

```python
@pytest.mark.integration
@pytest.mark.api
class TestHealthAPI:
    def test_health_check(self, client):
        """测试健康检查接口"""
        response = client.get('/api/health')
        assert response.status_code == 200
        data = response.get_json()
        assert data['success'] is True
```

**覆盖范围**：
- API 路由端点（请求/响应格式）
- 凭证验证完整流程
- 配置映射完整流程
- 迁移执行完整流程

### 3. 测试标记

| 标记 | 描述 | 用途 |
|------|------|------|
| `@pytest.mark.unit` | 单元测试 | 测试独立模块 |
| `@pytest.mark.integration` | 集成测试 | 测试模块协作 |
| `@pytest.mark.aliyun` | 阿里云相关 | 测试阿里云功能 |
| `@pytest.mark.tencent` | 腾讯云相关 | 测试腾讯云功能 |
| `@pytest.mark.mapper` | 映射引擎 | 测试配置映射逻辑 |
| `@pytest.mark.migration` | 迁移执行 | 测试迁移流程 |
| `@pytest.mark.api` | API 路由 | 测试 REST API 端点 |
| `@pytest.mark.slow` | 慢速测试 | 运行时间较长的测试 |

## 运行测试

### 使用测试脚本（推荐）

```bash
# 运行所有测试
./run_tests.sh

# 运行单元测试
./run_tests.sh unit

# 运行集成测试
./run_tests.sh integration

# 运行阿里云相关测试
./run_tests.sh aliyun

# 运行腾讯云相关测试
./run_tests.sh tencent

# 运行映射引擎测试
./run_tests.sh mapper

# 运行迁移测试
./run_tests.sh migration

# 生成覆盖率报告
./run_tests.sh coverage

# 快速测试（跳过慢速测试）
./run_tests.sh quick

# 清理测试缓存
./run_tests.sh clean
```

### 使用 pytest 命令

```bash
# 运行所有测试
pytest

# 运行指定文件
pytest tests/unit/test_mapper.py

# 运行指定测试类
pytest tests/unit/test_mapper.py::TestProtocolMapping

# 运行指定测试方法
pytest tests/unit/test_mapper.py::TestProtocolMapping::test_http_mapping

# 按标记运行
pytest -m unit              # 仅单元测试
pytest -m integration       # 仅集成测试
pytest -m "not slow"        # 跳过慢速测试

# 详细输出
pytest -v

# 显示 print 输出
pytest -s

# 失败时停止
pytest -x

# 失败时进入调试
pytest --pdb
```

## 编写测试

### 1. 使用 Fixture

在 `conftest.py` 中定义全局 fixture：

```python
import pytest
from app import create_app

@pytest.fixture
def app():
    """创建 Flask 测试应用"""
    app = create_app(testing=True)
    return app

@pytest.fixture
def client(app):
    """创建 Flask 测试客户端"""
    return app.test_client()

@pytest.fixture
def mock_aliyun_config():
    """Mock 阿里云配置"""
    return {
        'access_key_id': 'test_ak_id',
        'access_key_secret': 'test_ak_secret',
        'region': 'cn-shanghai'
    }

@pytest.fixture
def mock_lb_data():
    """Mock 阿里云 CLB 实例数据"""
    return {
        'LoadBalancerId': 'lb-test123',
        'LoadBalancerName': 'test-lb',
        'Address': '1.2.3.4',
        'AddressType': 'internet',
        'LoadBalancerStatus': 'active'
    }
```

### 2. Mock 外部 API 调用

```python
from unittest.mock import patch

@patch('app.services.aliyun.client.AliyunSLBClient.describe_load_balancers')
def test_list_instances(mock_api, service, mock_lb_data):
    """测试拉取 CLB 实例列表"""
    mock_api.return_value = {
        'LoadBalancers': {'LoadBalancer': [mock_lb_data]},
        'TotalCount': 1
    }
    result = service.list_instances('cn-shanghai')
    assert len(result) == 1
    assert result[0]['id'] == 'lb-test123'
```

### 3. 测试配置映射

```python
@pytest.mark.unit
@pytest.mark.mapper
class TestSchedulerMapping:
    """调度算法映射测试"""

    @pytest.mark.parametrize("aliyun_scheduler,expected_tencent", [
        ('wrr', 'WRR'),
        ('wlc', 'LEAST_CONN'),
        ('rr', 'WRR'),
    ])
    def test_scheduler_mapping(self, aliyun_scheduler, expected_tencent):
        result = SchedulerConverter.convert(aliyun_scheduler)
        assert result == expected_tencent

    def test_unsupported_scheduler(self):
        result, warning = SchedulerConverter.convert_with_warning('ch')
        assert result == 'WRR'  # 降级为默认值
        assert '不兼容' in warning
```

### 4. 测试异常处理

```python
def test_invalid_credential(service):
    """测试无效凭证处理"""
    with pytest.raises(ValueError) as exc_info:
        service.verify_credential('', '')
    assert '凭证不能为空' in str(exc_info.value)
```

## 测试最佳实践

### 1. 测试命名

```python
# ✅ 好的命名 — 描述清晰
def test_list_instances_returns_empty_when_no_clb_exists():
def test_protocol_mapping_converts_http_to_uppercase():
def test_migration_resumes_from_checkpoint_after_interruption():

# ❌ 不好的命名 — 含义不清
def test1():
def test_mapper():
```

### 2. Mock 策略

```python
# ✅ Mock 外部依赖（云 API 调用）
@patch('app.services.aliyun.client.AliyunSLBClient.describe_load_balancers')
def test_with_mock(mock_api):
    mock_api.return_value = {...}

# ✅ 使用 pytest-mock 的 mocker fixture
def test_with_mocker(mocker):
    mock_api = mocker.patch('app.services.aliyun.client.describe_load_balancers')
```

### 3. 测试隔离

- 每个测试独立运行，不依赖执行顺序
- 使用 fixture 提供干净的测试环境
- 避免测试间共享可变状态
- 使用 `yield` fixture 做清理

```python
@pytest.fixture
def temp_data_dir(tmp_path):
    """提供临时数据目录"""
    data_dir = tmp_path / "data"
    data_dir.mkdir()
    yield data_dir
    # 自动清理（tmp_path 由 pytest 管理）
```

### 4. 持续集成

```bash
# CI/CD 中运行测试
pytest -v --cov=app --cov-report=xml

# 检查覆盖率是否达标
pytest --cov=app --cov-fail-under=70
```

## 重点测试场景

### 配置映射引擎测试

| 测试项 | 场景 |
|--------|------|
| 协议映射 | TCP/UDP/HTTP/HTTPS 正确映射 |
| 调度算法映射 | wrr→WRR、wlc→LEAST_CONN、rr→WRR、ch→WRR（降级） |
| 健康检查映射 | 协议、端口、路径、间隔、阈值字段映射 |
| 转发规则映射 | 域名、URL 路径正确映射 |
| 不兼容检测 | 不支持的配置项正确标记 |
| 端口冲突检测 | 多对一合并时检测端口冲突 |

### 迁移执行引擎测试

| 测试项 | 场景 |
|--------|------|
| 正常执行 | 所有项成功完成 |
| 部分失败 | 单项失败不阻塞整体 |
| 断点续传 | 中断后从上次位置继续 |
| 暂停/继续 | 手动暂停后可继续 |
| 状态持久化 | 每步完成后状态正确写入 |

### 安全测试

| 测试项 | 场景 |
|--------|------|
| 阿里云只读 | 确保不调用任何写入 API |
| 凭证脱敏 | 日志中不包含 AK/SK |
| 二次确认 | 写操作需要用户确认 |

## 常见问题

### Q: 测试找不到模块？
```bash
# 确保激活虚拟环境
source venv/bin/activate
pip install -r requirements.txt
```

### Q: Mock 不生效？
```python
# 确保 patch 的路径是使用的地方，不是定义的地方
# ❌ 错误
@patch('app.services.aliyun.client_factory.AliyunClientFactory')

# ✅ 正确
@patch('app.services.aliyun.service.AliyunClientFactory')
```

### Q: 如何调试单个测试？
```bash
# 运行单个测试，显示详细输出
pytest tests/unit/test_mapper.py::TestProtocolMapping::test_http_mapping -v -s

# 失败时进入交互式调试
pytest tests/unit/test_mapper.py -x --pdb
```
