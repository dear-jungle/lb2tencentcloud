# 开发环境热重载指南

## 概述

本项目已配置完整的代码热重载功能，在开发模式下修改代码文件后**无需手动重启服务**即可生效。

---

## 快速开始

### 1. 确认开发模式

检查 `.env` 文件：
```bash
DEV_MODE=1
```

### 2. 启动开发服务器

```bash
./start.sh
```

---

## 热重载机制

### Python 文件 (.py)

| 项 | 说明 |
|---|------|
| 监控目录 | `app/`、`server.py` |
| 工作原理 | Gunicorn `--reload` 选项监测文件变化 |
| 生效时间 | 2-3 秒自动重启 |

**操作**：修改 `.py` 文件 → 保存 → 等待 2-3 秒 → 自动生效

### 前端文件 (HTML/CSS/JS)

| 项 | 说明 |
|---|------|
| 监控目录 | `public/` |
| 工作原理 | 开发模式禁用浏览器缓存 |
| 生效时间 | 即时（刷新浏览器） |

**操作**：修改文件 → 保存 → F5 刷新浏览器

### Vite HMR 热更新（可选）

```bash
cd frontend && npm run dev
```

启用 Vite 开发服务器后，修改 JS/CSS 文件会自动热更新，无需手动刷新。

### 配置文件 (.env)

| 项 | 说明 |
|---|------|
| 监控文件 | `.env` |
| 工作原理 | Gunicorn `--reload-extra-file` |
| 生效时间 | 2-3 秒自动重启 |

## 典型开发流程

```bash
# 1. 启动开发服务器（只需一次）
./start.sh

# 2. 打开实时日志（另一个终端）
tail -f logs/app.log

# 3. 编辑代码
# Python 文件：保存后 2-3 秒自动重启
# 前端文件：保存后刷新浏览器

# 4. 浏览器验证效果

# 5. 停止服务
pkill -f "gunicorn.*server:app"
```

## 生产部署

切换到生产模式：

```bash
# 方式一：修改 .env
DEV_MODE=0

# 方式二：使用生产模式启动
./start.sh --prod --build
```

**生产模式特性**：
- 禁用自动重载（节省资源）
- 启用缓存（提升性能）
- 4 个 Worker 进程（并发处理）
- Vite 构建打包（代码压缩 + hash 指纹）
- Hypercorn HTTP/2 支持

## 常见问题

### Q: Python 代码修改后没有自动重启？
1. 确认 `DEV_MODE=1`
2. 查看日志：`tail -f logs/app.log`
3. 检查 Python 语法错误（语法错误会阻止重启）
4. 手动重启：`./start.sh restart`

### Q: 前端修改后刷新看不到变化？
1. 使用硬刷新：`Ctrl + F5`
2. 打开开发者工具（F12），勾选 "Disable cache"
3. 确认修改了正确的文件（`public/` 目录下）
