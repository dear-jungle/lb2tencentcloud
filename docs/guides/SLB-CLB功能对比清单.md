## 二、详细功能对比

### 2.1 监听器管理

|--------|-----------|-----------|--------|----------|
| **四层监听** | | | | |
| TCP | ✅ 支持 | ✅ 支持 | ✅ 完全兼容 | 直接映射 |
| UDP | ✅ 支持 | ✅ 支持 | ✅ 完全兼容 | 直接映射 |
| **七层监听** | | | | |
| HTTP | ✅ 支持 | ✅ 支持 | ✅ 完全兼容 | 直接映射 |

| gRPC | ❌ 不支持 | ✅ 支持 | - | 腾讯云扩展功能 |

#### 2.1.2 调度算法

> **官方文档说明**:
> - 阿里云传统型 CLB 支持的调度算法：
>   - **轮询 (RR)**: 按访问顺序依次分发请求到后端服务器，适用于短连接服务
>   - **加权轮询 (WRR)**: 根据权重值轮询，权重越高被选中概率越大
>   - **一致性哈希 (CH)**: 支持源IP哈希、四元组哈希、QUIC ID哈希
> - 腾讯云 CLB 支持: 加权轮询 (WRR)、加权最小连接数 (WLC)、源地址散列 (IP Hash)

| 功能项 | 阿里云传统型 CLB | 腾讯云 CLB | 兼容性 | 映射说明 |
|--------|-----------------|-----------|--------|----------|
| 加权轮询 | `wrr` | `WRR` | ✅ 完全兼容 | 直接映射 |
| 加权最小连接数 | `wlc` | `LEAST_CONN` | ✅ 完全兼容 | 直接映射 |
| 轮询 (等权重) | `rr` | `WRR` | ✅ 自动转换 | 权重均等时等效 |
| 源地址哈希 (sch) | ✅ 支持 | `IP_HASH` | ✅ 可映射 | 腾讯云 IP Hash 等效于阿里云 sch |
| 四元组哈希 (tch) | ✅ 支持 | - | ❌ 不支持 | 腾讯云 CLB 不支持 |
| QUIC ID 哈希 | ✅ 支持 | - | ❌ 不支持 | 腾讯云 CLB 不支持 |

#### 2.1.3 会话保持

> **官方文档说明**:
> - 阿里云传统型 CLB 会话保持配置：
>   - **四层 (TCP/UDP)**: 基于源IP哈希，超时时间最长 3600 秒
>   - **七层 (HTTP/HTTPS)**: 基于 Cookie，支持植入 Cookie 和重写 Cookie 两种模式，超时时间最长 86400 秒
>   - **注意**: 阿里云传统型 CLB **不支持** Header 模式和 HMAC 认证模式（仅 ALB 支持）
> - 腾讯云 CLB 会话保持支持: Cookie 模式 (NORMAL/CUSTOMIZED)、IP Hash 模式 (四层)
> - **重要**: 腾讯云 CLB 的 `LEAST_CONN` (最小连接数) 算法与会话保持功能**不能同时开启**

| 功能项 | 阿里云传统型 CLB | 腾讯云 CLB | 兼容性 | 映射说明 |
|--------|-----------------|-----------|--------|----------|
| 开启/关闭 | `on`/`off` | `SessionExpireTime > 0` | ✅ 完全兼容 | 直接映射 |
| 植入 Cookie | `insert` (默认) | `NORMAL` | ✅ 完全兼容 | 直接映射 |
| 重写 Cookie | `server` | `CUSTOMIZED` | ✅ 完全兼容 | 直接映射 |
| Cookie 名称 | `SERVERID` (默认) | `JSESSIONID` (默认) | ⚠️ 默认值不同 | 可配置相同 |
| Cookie 超时 | 最长 86400 秒 | `SessionExpireTime` | ✅ 完全兼容 | 直接映射 |
| 基于 Header | ❌ 不支持 | - | - | 仅 ALB 支持 |
| 基于 HMAC 认证 | ❌ 不支持 | - | - | 仅 ALB 支持 |
| IP Hash (四层) | `sch` | `IP_HASH` | ✅ 完全兼容 | 四层监听直接映射 |

#### 2.1.4 健康检查

> **官方文档说明**:
> - 阿里云传统型 CLB 健康检查：
>   - **协议支持**: TCP、UDP、HTTP 三种检查协议
>   - **协议对应关系**: TCP监听支持TCP/HTTP检查，UDP监听支持TCP/UDP/HTTP检查，HTTP/HTTPS监听支持HTTP检查
>   - **HTTP检查方法**: 支持 HEAD（默认）和 GET
>   - **健康检查路径**: 建议使用专门的健康检查接口（如 `/health`），避免使用业务复杂页面
>   - **健康检查域名**: 如后端校验Host字段，必须配置此项
>   - **正常状态码**: 默认识别 `http_2xx` 和 `http_3xx`
>   - **UDP检查**: 支持配置请求内容和预期返回结果
> - 腾讯云 CLB 健康检查支持: TCP/HTTP 协议、检查路径、域名、间隔、超时、健康/不健康阈值
> - **注意**: 腾讯云 CLB 集群由多台物理机组成，每台物理机独立发送健康检查探测，可能导致后端收到的探测频率高于配置值

| **检查参数** | | | | |

### 2.2 转发规则 (七层)

> **官方文档说明**:
> - 阿里云传统型 CLB 转发规则（仅七层支持）：
>   - **域名转发**: 支持精确匹配和通配符匹配（`*.example.com`）
>   - **URL 路径转发**: 基于前缀最长匹配，如 `/abc` 和 `/abcd` 同时配置，请求 `/abcde` 优先匹配 `/abcd`
>   - **匹配优先级**: 精确匹配 > 小范围通配符 > 大范围通配符
>   - **数量限制**: 单个 HTTP/HTTPS 监听最多 40 条转发规则
>   - **后端组限制**: 转发规则仅支持虚拟服务器组
>   - **重定向**: 支持 HTTP 重定向至 HTTPS
>   - **WebSocket**: 原生支持 WS/WSS 协议
>   - **HTTP/2**: 原生支持，向下兼容 HTTP/1.x
> - 腾讯云 CLB 转发规则相对基础，高级功能需在后端实现

| 功能项 | 阿里云传统型 CLB | 腾讯云 CLB | 兼容性 | 映射说明 |
|--------|-----------------|-----------|--------|----------|
| 基于域名匹配 | ✅ 支持 | ✅ 支持 | ✅ 完全兼容 | 直接映射 |
| 基于 URL 路径匹配 | ✅ 支持 | ✅ 支持 | ✅ 完全兼容 | 直接映射 |
| 通配符域名 | ✅ 支持 | ✅ 支持 | ✅ 完全兼容 | 直接映射 |
| 正则表达式匹配 | ❌ 不支持 | ❌ 不支持 | - | 两者均不支持 |
| 基于 HTTP Header 匹配 | ❌ 不支持 | ❌ 不支持 | - | 仅 ALB 支持 |
| 基于查询字符串匹配 | ❌ 不支持 | ❌ 不支持 | - | 仅 ALB 支持 |
| 基于 Cookie 匹配 | ❌ 不支持 | ❌ 不支持 | - | 仅 ALB 支持 |
| 基于 HTTP 方法匹配 | ❌ 不支持 | ❌ 不支持 | - | 仅 ALB 支持 |
| 基于源 IP 匹配 | ❌ 不支持 | ❌ 不支持 | - | 仅 ALB 支持 |

### 2.3 访问控制 (ACL)

> **官方文档说明**:
> - 阿里云传统型 CLB 访问控制：
>   - 支持白名单和黑名单，灵活控制客户端访问
>   - 结合云盾提供 5 Gbps 的 DDoS 防护能力
>   - ACL 与安全组可同时使用
> - 腾讯云 CLB 主要通过**安全组**实现访问控制（四层/七层建议后端安全组全放通）
> - **重要**: 腾讯云 CLB 设置访问黑名单需利用安全组规则的**顺序匹配**特性

| 功能项 | 阿里云传统型 CLB | 腾讯云 CLB | 兼容性 | 映射说明 |
|--------|-----------------|-----------|--------|----------|
| 白名单 | `white` | 安全组放通 | ✅ 完全兼容 | 使用安全组替代 |
| 黑名单 | `black` | 安全组拒绝 | ✅ 完全兼容 | 使用安全组替代 |
| CIDR 规则 | ✅ 支持 | ✅ 支持 | ✅ 完全兼容 | 直接映射 |
| 规则注释 | `AclEntryComment` | - | ⚠️ 部分支持 | 仅用于审计，不影响功能 |
| 实例级 ACL | ✅ 支持 | 安全组 | ✅ 完全兼容 | 使用安全组替代 |
| 监听器级 ACL | ✅ 支持 | 安全组 | ✅ 完全兼容 | 使用安全组替代 |
| 规则顺序匹配 | - | ✅ 支持 | ✅ 完全兼容 | 自顶向下匹配 |
| DDoS 防护 | 5 Gbps (云盾) | 需单独购买 | ⚠️ 需额外配置 | 使用云安全产品 | |

### 2.4 安全策略 (HTTPS)

> **官方文档说明**:
> - 阿里云传统型 CLB HTTPS 支持：
>   - 统一证书管理，SSL 解密在 CLB 上进行，降低后端 CPU 开销
>   - 支持 SNI，可挂载多个证书，根据访问域名转发至不同后端服务器组
>   - 支持 SSL 协议版本配置
>   - 支持加密套件自定义
> - 腾讯云 CLB 支持标准的 SSL/TLS 配置

|--------|-----------------|-----------|--------|----------|

| 证书类型 | 通用/边缘/自定义 | 通用/边缘/自定义 | ✅ 完全兼容 | 直接映射 |
| SSL 协议版本 | TLS 1.0/1.1/1.2/1.3 | TLS 1.0/1.1/1.2/1.3 | ✅ 完全兼容 | 直接映射 |
| 加密套件 | ✅ 支持自定义 | ✅ 支持自定义 | ✅ 完全兼容 | 直接映射 |
| 证书优先级 | ✅ 支持 | ✅ 支持 | ✅ 完全兼容 | 直接映射 |
| SNI 多证书支持 | ✅ 支持 | ✅ 支持 | ✅ 完全兼容 | 直接映射 |

| 监听器 | 带宽峰值时间 | 四层/七层 | 使用腾讯云带宽包 |

### 3.2 需要调整的配置

| 功能类别 | 功能项 | 调整说明 |
|----------|--------|----------|
| 健康检查 | 检查间隔 | 阿里云 [1-60]s，腾讯云 [2-300]s，超出范围需调整 |
| 健康检查 | 超时时间 | 阿里云 [1-60]s，腾讯云 [2-60]s，超出范围需调整 |
| 健康检查 | HTTP 状态码 | 格式不同，需转换为位掩码 |
| 会话保持 | Cookie 名称 | 阿里云默认 `SERVERID`，腾讯云默认 `JSESSIONID` |

## 五、迁移注意事项

1. **调度算法迁移**: 
   - `sch` (源地址哈希) 可映射到腾讯云 `IP_HASH`
   - `tch`/`qch` 需评估业务影响后选择 `WRR` 或 `LEAST_CONN`
   - 注意: 腾讯云 `LEAST_CONN` 与会话保持**不能同时开启**

2. **会话保持迁移**: 
   - 阿里云传统型 CLB 仅支持 Cookie 模式（植入/重写），不支持 Header 和 HMAC 模式
   - 如使用默认 Cookie 名称 `SERVERID`，迁移后需调整为腾讯云默认的 `JSESSIONID` 或自定义相同名称
   - 四层监听的 `sch` 可直接映射到 `IP_HASH`

3. **健康检查迁移**: 
   - 需确保检查间隔和超时时间在腾讯云支持范围内
   - 注意: 腾讯云 CLB 集群多物理机探测，实际探测频率可能高于配置值
   - 注意: 腾讯云不支持 UDP 健康检查和 HEAD 请求方法
   - 建议创建专用 `/health` 接口用于健康检查

4. **转发规则迁移**: 
   - 阿里云传统型 CLB 仅支持域名匹配和 URL 路径匹配，不支持正则表达式
   - 转发规则数量限制: 阿里云 40 条/监听，需确保迁移后不超限

5. **监听器拆分**: HTTP/HTTPS 混合监听需拆分为独立监听器

6. **安全策略迁移**: 
   - DDoS 防护需单独购买云安全产品
   - ACL 功能使用安全组替代，注意规则顺序匹配

## 六、阿里云负载均衡产品线说明

> **重要**: 阿里云负载均衡包含三个产品线，功能支持有所不同：

| 产品线 | 定位 | 主要特性 |
|--------|------|----------|
| **ALB** (应用型) | 七层负载均衡 | HTTP/HTTPS/QUIC，丰富路由规则（Header/Cookie/查询字符串匹配、正则、URL重写、Header写入删除、QPS限速、CORS），WAF 集成，国密证书、mTLS |
| **NLB** (网络型) | 四层负载均衡 | TCP/UDP/TCPSSL，超高性能(亿级连接)，SSL 卸载 |
| **CLB** (传统型) | 综合型 | 四层+七层，基础功能，本文档对比对象 |

本文档对比的是阿里云 **传统型 CLB (Classic Load Balancer)** 与腾讯云 **CLB** 的功能差异。如使用阿里云 **ALB**，部分高级功能可能无法在腾讯云 CLB 中找到直接替代。
