# 配置映射改版：实例维度对比

## Problem

当前配置映射流程将所有源端实例的监听器**打平为一维列表**传递给映射引擎，丢失了实例维度信息。用户无法直观看到"A实例的哪些配置迁移到B实例上"，也缺少源端→目标端实例的关联步骤。映射结果以 JSON 对比展示，对非技术用户不友好。

具体问题：
- 步骤2（选择实例）只选源端，没有目标端关联
- 步骤3（配置映射）所有监听器扁平展示，无法区分归属哪个实例
- 不兼容项只展示原因，不提供替代方案选择
- 端口冲突检测后端已实现但前端未接入
- 可视化对比只有 JSON 格式，缺少字段级可读对比

## Solution

改版步骤2和步骤3的交互与数据流：

1. **步骤2 改为"实例关联"**：同一页面左右两栏，左栏选源端实例，右栏关联目标端实例（下拉选择已有实例），支持一对一和多对一
2. **步骤3 改为"配置对比"**：按实例分组的两层可视化对比（实例总览 → 监听器/转发规则字段级对比），JSON 对比放侧边抽屉作为辅助
3. 不兼容项自动推荐最佳替代方案（功能最接近优先），用户可通过下拉框修改
4. 多对一场景接入端口冲突检测，弹出列表让用户逐项选择处理方式

## Scope

### New Capabilities
- `instance-association`: 实例关联页面（源端↔目标端配对、多对一端口冲突检测）
- `visual-config-diff`: 可视化配置对比（实例分组、字段级对比、JSON 抽屉、替代方案推荐）

### Modified Capabilities
- `config-mapping-engine`: 映射引擎支持按实例分组映射，不兼容项增加推荐替代方案字段
- `step-wizard`: 步骤2改名"实例关联"，步骤3改名"配置对比"

## Impact

- 前端：`aliyun/index.js` 重写为实例关联页面，`mapping/index.js` 重写为配置对比页面，`plan/index.js` 小改支持实例分组
- 后端：`mapping_routes.py` 接口改为按实例分组传参，`engine.py` 增加推荐逻辑，`tencent_routes.py` 增加目标端实例列表接口
- 状态：`state-manager.js` 增加 `instanceMappings` 状态
- CSS：新增抽屉组件、字段对比卡片样式
