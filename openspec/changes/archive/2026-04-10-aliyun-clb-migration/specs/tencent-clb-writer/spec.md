## ADDED Requirements

### Requirement: 写操作二次确认弹窗
系统 SHALL 在执行腾讯云 CLB 的每一类写操作（创建监听器、创建转发规则、修改配置等）前弹出确认弹窗，展示操作详情，用户 MUST 点击确认后才执行。

#### Scenario: 创建操作二次确认
- **WHEN** 系统即将在腾讯云创建新的监听器或转发规则
- **THEN** 弹出确认弹窗，展示将要创建的资源详情，用户点击"确认"后执行，点击"取消"则跳过

#### Scenario: 修改/覆盖操作二次确认
- **WHEN** 系统即将修改或覆盖腾讯云已有的 CLB 配置
- **THEN** 弹出确认弹窗，明确展示当前值与将要覆盖的新值对比，用户确认后执行

### Requirement: 禁止静默改动
系统 SHALL NOT 在无用户确认的情况下对腾讯云 CLB 执行任何写入操作。每一个写操作 MUST 有明确的用户交互确认。

#### Scenario: 无确认不写入
- **WHEN** 系统需要执行腾讯云写操作
- **THEN** 如果用户未通过弹窗确认，系统不执行该操作，记录为"用户跳过"

### Requirement: 批量确认模式
系统 SHALL 提供"批量确认"选项，用户可选择一次性确认所有待执行的同类写操作，避免逐一弹窗。

#### Scenario: 用户选择批量确认
- **WHEN** 用户勾选"批量确认同类操作"
- **THEN** 系统对同类写操作（如批量创建监听器）只弹一次确认弹窗，展示所有操作列表

### Requirement: 拉取腾讯云已有配置
系统 SHALL 在写入前拉取腾讯云目标 CLB 的现有配置，检测冲突（如端口已占用、规则已存在）。

#### Scenario: 检测到配置冲突
- **WHEN** 目标腾讯云 CLB 已存在与待迁移配置冲突的监听器或规则
- **THEN** 系统在确认弹窗中明确提示冲突详情，让用户选择覆盖、跳过或新建

### Requirement: 写操作错误处理
系统 SHALL 捕获腾讯云 API 写入错误，展示错误信息，并允许用户选择重试或跳过。

#### Scenario: 写入失败
- **WHEN** 腾讯云 API 调用返回错误
- **THEN** 系统展示错误信息，提供"重试"和"跳过"按钮

### Requirement: 自动创建 CLB 实例
系统 SHALL 支持在"自动创建新实例"模式下，通过腾讯云 API 创建新的 CLB 实例。用户 MUST 填写完整参数（实例名称、网络类型、VPC、子网、所属项目），创建操作 MUST 经用户二次确认。

#### Scenario: 用户选择自动创建实例
- **WHEN** 用户选择"自动创建新实例"模式
- **THEN** 系统展示完整的实例创建表单（名称、网络类型公网/内网、VPC 选择、子网选择、所属项目），用户填写后弹出二次确认弹窗展示全部参数，确认后调用 CreateLoadBalancer API 创建

#### Scenario: 使用已有实例
- **WHEN** 用户选择"使用已有实例"模式
- **THEN** 系统展示腾讯云目标地域下的 CLB 实例列表供选择

### Requirement: 高级参数和策略配置写入
系统 SHALL 将映射后的高级参数（连接超时、空闲超时、会话保持）和策略配置（带宽限制、访问控制）写入腾讯云 CLB。所有写操作 MUST 经用户二次确认。

#### Scenario: 高级参数写入
- **WHEN** 迁移计划包含高级参数配置
- **THEN** 系统在二次确认弹窗中展示高级参数详情，确认后写入腾讯云

#### Scenario: 策略配置写入
- **WHEN** 迁移计划包含访问控制策略或带宽限制
- **THEN** 系统在二次确认弹窗中展示策略配置详情，确认后写入腾讯云
