## Why

企业在多云环境或云迁移场景下，需要将阿里云 CLB（经典负载均衡）的配置和规则迁移到腾讯云 CLB。手动迁移配置繁琐、易出错，且涉及监听器规则、转发策略、健康检查等大量细节。需要一个可视化的 Web 工具，让用户通过浏览器以分步引导的方式完成迁移，降低操作门槛，提升迁移效率和准确性。

当前阶段聚焦于 CLB 配置和规则的迁移，后续再扩展到 LB 实例映射、SSL 证书迁移及后端服务迁移。

## What Changes

- 新增 Web 应用，提供浏览器可访问的分步操作界面
- 支持连接阿里云（只读）和腾讯云（受控写入）API
- 实现阿里云 CLB 配置的读取、解析和展示
- 实现配置映射引擎：将阿里云 CLB 配置映射为腾讯云 CLB 对应配置
- 支持迁移实例的一对一、一对多、多对一映射关系
- 对无法映射或不支持的配置项，迁移前明确提醒用户
- 目标端（腾讯云）的任何修改/覆盖操作必须弹窗提示并二次确认，严禁静默改动
- 源端（阿里云）全程只读，禁止任何修改、删除或覆盖操作
- 迁移过程展示实时进度条和操作日志
- 支持意外中断后从断点续传
- 迁移完成后生成清晰的迁移报告（成功项、失败项、跳过项、不兼容项）
- 整体流程拆分为清晰的多个步骤，用户可明确看到当前进度及剩余步骤

## Capabilities

### New Capabilities

- `cloud-credentials`: 云平台凭证管理，支持阿里云和腾讯云 AK/SK 的安全输入、验证和连接测试
- `aliyun-clb-reader`: 阿里云 CLB 配置只读读取，包括实例列表、监听器、转发规则、健康检查、后端服务器组等配置的拉取和解析
- `config-mapping-engine`: 配置映射引擎，将阿里云 CLB 配置映射为腾讯云 CLB 配置，处理字段差异、枚举值转换，识别不兼容项
- `migration-plan`: 迁移计划生成与预览，展示映射关系（一对一/一对多/多对一）、不兼容提醒、用户确认
- `tencent-clb-writer`: 腾讯云 CLB 配置受控写入，所有写操作必须经用户二次确认弹窗，禁止静默改动
- `migration-execution`: 迁移执行引擎，实时进度展示、操作日志、断点续传能力
- `migration-report`: 迁移报告生成，汇总成功/失败/跳过/不兼容项，支持查看和导出
- `step-wizard`: 分步向导 UI，将整体流程拆分为清晰步骤，展示当前进度和剩余步骤

### Modified Capabilities

（无，本次为全新项目）

## Impact

- **前端**：新建 Web 应用，需选择前端框架（React/Vue 等），实现分步向导、表单、弹窗确认、进度条、日志面板等 UI 组件
- **后端**：新建 API 服务，集成阿里云 SLB SDK（只读）和腾讯云 CLB SDK（受控读写），实现配置读取、映射、写入、断点续传等业务逻辑
- **安全**：AK/SK 凭证的传输和存储安全；源端只读权限控制；目标端写操作二次确认机制
- **依赖**：阿里云 SLB/CLB OpenAPI SDK、腾讯云 CLB API SDK
- **数据**：迁移任务状态持久化（支持断点续传）、迁移日志和报告存储
- **部署**：Docker 容器化部署

## 需求澄清结论

以下为经过需求澄清后确认的决策：

| # | 问题 | 决策 |
|---|------|------|
| 1 | 阿里云 CLB 范围 | **先做 CLB（经典负载均衡）**，ALB 后续再考虑 |
| 2 | 腾讯云目标端操作 | **支持两种模式**：使用已有实例 或 自动创建新实例（用户选择） |
| 3 | 本阶段迁移范围 | 监听器 + 转发规则 + 健康检查 + 会话保持 + **连接超时/空闲超时等高级参数** + **带宽限制、访问控制等策略配置** |
| 4 | 系统使用模式 | **单用户**，凭证可保存到 .env 供复用 |
| 5 | 实例映射关系 | **仅支持一对一和多对一**，一对多暂不支持 |
| 6 | 地域范围 | **仅中国大陆地域** |
| 7 | 部署方式 | **Docker 部署** |
| 8 | 迁移历史记录 | **全部保留，不自动清理** |
| 9 | 凭证安全方案 | 运行时存内存，**允许用户选择保存到 .env 供复用**（.env 不入库） |
| 10 | 自动创建实例参数 | 用户**填写完整参数**（名称、网络类型、VPC、子网、项目等） |
| 11 | 多对一端口冲突 | 冲突时**展示列表**，用户逐项选择（覆盖/跳过/重命名端口） |
| 12 | 单项失败行为 | 用户可在执行前选择模式：**"失败继续"或"失败暂停"** |
