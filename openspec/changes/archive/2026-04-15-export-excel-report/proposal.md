# 导出报告改为 Excel 格式（面向使用者）

## Problem

当前迁移报告只支持 JSON 格式导出，JSON 是给开发者看的数据结构，对业务人员完全不友好：
- 字段名是英文技术术语（`source_config`、`incompatible_items` 等）
- 没有中文说明，看不懂每一项的含义
- 没有结论性摘要，看不出"这次迁移整体成功还是失败了"
- 数据都堆在一起，缺乏层次结构

实际使用场景：迁移完成后，业务负责人或运维负责人需要存档一份"迁移报告"，用于：
1. 复盘确认：哪些配置迁移成功，哪些需要手动处理
2. 留档审计：记录迁移时间、操作人、涉及实例
3. 向上汇报：让管理层看得懂的一份汇总

## Solution

将导出格式改为 Excel（`.xlsx`），用多 Sheet 结构组织内容，面向不同受众：

| Sheet | 受众 | 内容 |
|-------|------|------|
| **封面·迁移摘要** | 所有人 | 迁移时间、账号信息、整体结论、成功率 |
| **实例迁移概览** | 业务/运维 | 每对实例（源→目标）的整体状态，一行一个实例 |
| **监听器配置明细** | 运维 | 每个监听器的迁移详情，用中文字段名 |
| **不兼容项说明** | 运维/技术 | 哪些配置无法迁移、原因、建议处理方式 |
| **操作日志** | 技术人员 | 完整执行日志，用于排查问题 |

同时保留 JSON 导出按钮（给技术人员），Excel 作为主要导出格式。

## Scope

### Modified Capabilities
- `migration-report`: 新增 Excel 导出功能，改版报告页面

## Impact

- 前端：`report/index.js` 新增 Excel 导出逻辑（使用 SheetJS / xlsx 库）
- 后端：报告页面无后端变化，Excel 由前端纯浏览器端生成
- 依赖：新增前端库 `xlsx`（SheetJS，纯 JS，无需后端）
