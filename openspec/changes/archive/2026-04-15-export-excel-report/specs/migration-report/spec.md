# migration-report Specification (Delta)

## MODIFIED Requirements

### Requirement: 报告导出支持 Excel 格式（主格式）
系统 SHALL 将 Excel（`.xlsx`）作为主要导出格式，生成面向业务人员可读的多 Sheet 报告。JSON 导出保留作为辅助格式。

#### Scenario: 导出 Excel 报告
- **WHEN** 用户点击"导出 Excel 报告"按钮
- **THEN** 系统在浏览器端生成并下载 `.xlsx` 文件，文件名含迁移日期，内含以下 Sheet

#### Scenario: Excel 报告 Sheet 结构
- **WHEN** 用户打开导出的 Excel 文件
- **THEN** 文件包含 5 个 Sheet：
  1. **封面·迁移摘要**：迁移时间、源端账号/地域、目标端地域、整体结论文字（"全部迁移成功"/"存在未完成项"）、成功率
  2. **实例迁移概览**：每对实例一行（源实例名、目标实例名、监听器总数、成功数、失败数、整体状态）
  3. **监听器配置明细**：每个监听器一行，列名用中文（所属实例、监听协议、监听端口、调度算法、健康检查、会话保持、迁移状态、备注）
  4. **不兼容项说明**：列出所有无法自动迁移的配置（配置名称、原值、不兼容原因、建议处理方式）
  5. **操作日志**：完整执行日志，供技术人员排查问题

#### Scenario: 中文字段名与业务可读性
- **WHEN** 用户查看 Excel 明细 Sheet
- **THEN** 所有列标题使用中文业务术语，状态值用中文（成功/失败/跳过/不兼容），不出现英文技术术语

#### Scenario: Excel 样式
- **WHEN** 用户打开 Excel
- **THEN** 标题行加粗底色（深蓝白字），成功行绿色背景，失败行红色背景，不兼容行橙色背景，数字列右对齐
